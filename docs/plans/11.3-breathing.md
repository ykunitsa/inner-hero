# §11.3 — Дыхание: план реализации

Рабочий документ шага §11.3. Источник истины остаётся `docs/redesign-spec.md` (§4 —
дыхание, §1 — принципы); здесь зафиксированы решения, которых в спеке нет, и то,
как §4 ложится на текущий код и дизайн-систему.

Спроектировано 21.07.2026. Три экрана + правило лестницы.

---

## 1. Что строим

Отдельный флоу из трёх экранов (**до → сессия → после**) плюс правило лестницы как
чистая логика. Дыхание по модели applied relaxation (Öst) — **тренировка навыка по
расписанию, не скорая помощь**; успех = сокращение времени до расслабления.

Общего шаблона с экспозицией нет и не будет (принцип 1.10): своя вью-модель, свои
экраны. Общее только дизайн-система и единый лог сессий.

**Сознательно не строим в этом шаге:**

- карточку статьи «у дверей» до первой сессии (часть 1.7) — приезжает с оболочкой в §11.6;
- ленту истории, экран «Где ты сейчас», карточку правила в Истории — §11.6;
- расписание и напоминания дыхания — оболочка;
- озвучку — §11.8;
- переезд экспозиционных слайдеров на новую линейку-диск — отдельная задача, см. §9.

Плитка «Дыхание» на «Упражнениях» становится активной, подпись меняется с
«Coming back soon» на корректирующую фразу спеки 2.2.

## 2. Журнал решений

Решения, принятые при проектировании; каждое — либо детализация спеки, либо
осознанное отступление. Порядок — как обсуждались.

| # | Решение | Обоснование |
|---|---|---|
| 1 | **Ритмичное = 4-4 без задержек** | Спека называет тип, но не раскрывает. Три паттерна тогда честно различаются: с задержками / с удлинённым выдохом / ровный |
| 2 | **Правило лестницы показывается на экране «До»** (временно) | Спека селит его в Историю (§2.3 п.2), но История — заглушка до §11.6. «До» — то место, где длительность и выбирается. В §11.6 канонический дом — карточка в Истории; тогда решаем, переносить или оставлять дубль (по умолчанию — переносить) |
| 3 | **Флага «предложение показано» нет** | Правило считается из истории на лету. Следствие: если серия продолжается (6 подряд), строка появится снова — формально это не «исчезло до следующего срабатывания», но в духе 1.7 («никаких флагов, только счётчик»). Заведём флаг, если начнёт раздражать на устройстве |
| 4 | **Хаптика: рампа = «делай», тишина = «держи»** | См. §5. Рампа появляется ровно там, где тишина была бы двусмысленной |
| 5 | **Описания типов — свойство, а не обещание** | «Для чего подходит» = рекомендация = нарушение 1.1 и 1.3. Граница: описываем *что это и насколько трудно*, никогда *что это даст* и *когда применять* |
| 6 | **Длительность — прокручиваемая линейка-диск**, а не системный слайдер | USAGE §Шкалы говорит для одного ординального значения брать системный слайдер. Осознанное отступление: автор хочет фактуру диска (референс Tide). Цена названа: 8 ступеней помещаются на экран целиком, прокрутка даёт ощущение, а не решает проблему |
| 7 | **Тип дыхания — видимые карточки, а не «сменить» списком** | Спека предлагает прятать за «сменить». Видимый ряд лучше отвечает 1.2 (между иконкой и действием нет меню) и стоит не тапов, а вертикали. Дефолт — последний использованный, выбор не блокирует старт |
| 8 | **Экран сессии не гаснет** — `isIdleTimerDisabled` | UIKit запрещён CLAUDE.md; здесь записанное исключение. Без него iOS гасит экран через 30 с…2 мин, а вместе с ним **останавливается CoreHaptics** — упражнение обрывается молча в обоих каналах. SwiftUI-эквивалента на iOS 26 нет |
| 9 | **Экраны «Сессия» и «После» — на тёмной поверхности** | 15 минут светло-серого экрана в глаза вечером; так же делают Watch Breathe, Tide, Calm. Отступление от USAGE («фон внутри флоу не меняется») — правило писалось про шаги формы, не про экран упражнения. Требует нового токена и тёмных вариантов контролов |
| 10 | **Круг — «плавающая клякса»**: размытие + морфинг формы + дрейф | Запрос автора по референсу. Жёсткое ограничение: морфинг ≤ 15% от дельты масштаба фаз — круг остаётся инструкцией, декорация не съедает сигнал |
| 11 | **Уход в фон паузу не ставит** | `wasPaused` по спеке означает «пользователь воспользовался паузой». Выставлять его за пользователя — врать в данных. Часы идут по стенным |
| 12 | **«Закончить» без удержания** (в отличие от экспозиции) | Там удержание 3 с защищает от тапа в кармане. Здесь экран включён, телефон в руке, кнопка в нижней капсуле, и досрочный выход **ничего не разрушает** — запись сохраняется, пользователь идёт на «После» |
| 13 | **Строки правила нет на экране «После»** | Момент завершения серии соблазнительно поймать там, но предложение сразу после практики превращает спокойный выход в подталкивание |

## 3. Данные

### `Models/BreathingSession.swift`

```swift
enum BreathingPattern: String, CaseIterable {
    // Persisted rawValues — never rename (CLAUDE.md).
    case box        // 4-4-4-4
    case fourSix    // вдох 4, выдох 6
    case rhythmic   // вдох 4, выдох 4
}

@Model final class BreathingSessionEntry {
    var createdAt: Date
    var patternRaw: String
    var plannedDurationSeconds: Int   // ступень лестницы, выбранная на «До»
    var actualDurationSeconds: Int?   // nil = сессия начата и не завершена
    var didRelax: Bool?               // nil = экран «После» не отвечен
    var wasPaused: Bool               // спека §4: факт паузы пишется в сессию
    var note: String?
}
```

Запись **вставляется в стор на «Начать»**, как в планируемой экспозиции: убийство
приложения посреди сессии оставляет правдивую частичную запись, а не пустоту (1.5).
`actualDurationSeconds` пишется на завершении, `didRelax`/`note` — на «После».

Регистрация в `Schema` в `App/Inner_HeroApp.swift`. Версионных схем нет —
предрелизная фаза, модель правится на месте, стор при необходимости пересоздаётся.

## 4. Правило лестницы

### `Models/BreathingLadder.swift` — чистый тип, без SwiftData

Фиксированные ступени (секунды): `900, 600, 420, 300, 180, 120, 60, 30`
(15 → 10 → 7 → 5 → 3 → 2 → 1 → 0.5 мин). Спуск = успех.

```
suggestion(history:pattern:currentDuration:) -> Suggestion?
    case stepDown(seconds)   // 5 подряд «успел» → «Попробовать N мин?»
    case stepUp(seconds)     // 2 подряд «не успел» → «Вернуться на N мин?»
```

Как считается:

1. Берём сессии **этого паттерна**, сортируем по `createdAt` убыв.
2. Берём префикс, где `plannedDurationSeconds == currentDuration`. Сессия на другой
   длительности **обрывает серию** — спека говорит «подряд на текущей длительности».
3. Сессии с `didRelax == nil` (не отвечено) **пропускаются, серию не рвут** — они не
   несут ответа, а не несут отрицательный ответ.
4. 5 подряд `true` → `stepDown` на следующую ступень вниз. 2 подряд `false` →
   `stepUp` на ступень вверх.
5. На краях лестницы (15 мин вверх, 0.5 мин вниз) предлагать нечего → `nil`.

Правило **никогда не меняет длительность само** (1.8) — только возвращает
предложение. Считается раздельно по типам дыхания.

## 5. Движок фаз и хаптика

### Паттерны

| Паттерн | Вдох | Задержка | Выдох | Задержка |
|---|---|---|---|---|
| Квадратное | 4 | 4 | 4 | 4 |
| 4-6 | 4 | — | 6 | — |
| Ритмичное | 4 | — | 4 | — |

Движок — **чистая функция** `phase(at elapsed:) -> (BreathPhase, progressInPhase)`.
Время инжектится, часы внутри логики не читаются.

### Грамматика хаптики

| Паттерн | Вдох | Задержка | Выдох | Задержка |
|---|---|---|---|---|
| Квадратное | рампа ↗ | тишина | рампа ↘ (зеркальная) | тишина |
| 4-6 | рампа ↗ | — | тишина | — |
| Ритмичное | рампа ↗ | — | тишина | — |

Правило, из которого это выведено: **рампа появляется ровно там, где тишина была бы
двусмысленной.** Без задержек тишина однозначно значит «выдох» — хватает модели Apple
Watch Breathe. С задержками тишина означала бы и выдох, и задержку, поэтому выдох
забирает рампу, а тишина остаётся только за задержками. Тапов на границах фаз нет
нигде.

- **Вдох:** начинается слабо, быстро усиливается, к концу ослабевает и «замедляется».
- **Выдох (только квадратное):** зеркально — начинается сильно, затухает. Иначе с
  закрытыми глазами вдох и выдох на ощупь неразличимы.

Реализация — `CHHapticEngine` (CoreHaptics) с continuous event и parameter curve.
`UIImpactFeedbackGenerator` такую рампу не даёт. Кривая интенсивности выносится
**чистой функцией** `intensity(phase:progress:) -> Float` и покрывается тестами;
движок — тонкая обёртка с фолбэком на дискретные импульсы, если engine недоступен.

> ⚠️ CoreHaptics **не работает в симуляторе**. Хаптика проверяется только на
> устройстве. Зазор между затуханием рампы и следующей фазой (~100 мс) тюнится там же.

## 7. Экран «Сессия»

**Работа:** вести дыхание по фазам до конца. Главный элемент — круг; единственное
регулярное действие — пауза.

```
        Квадратное · 10 мин                  MetaLabel, тихая ориентировка

              ╭───────────╮
             │             │                 КЛЯКСА — единственный акцент, positive
              ╰───────────╯                  растёт/сжимается, ЗАМИРАЕТ на задержках

                 Вдох                        .h2, подпись фазы ПОД кругом

   ┌───────────────────────────────────┐
   │   ⏹     │    07:12    │     ⏸     │     SessionFlowBottomPill (тёмный вариант)
   └───────────────────────────────────┘     закончить · осталось · пауза
```

**Фон:** `AppColors.sessionSurface` (тёмный в обеих темах). Переход со «До» —
кроссфейд `AppAnimation.slow`, не рез.

**Решения по раскладке:**

- Подпись фазы **под** кругом, не внутри: круг масштабируется, текст внутри прыгал бы в размере.
- Обратный отсчёт **мелкий, в капсуле** (`.mono`), не `.timerDisplay` по центру: содержимое
  экрана — круг, крупный таймер отобрал бы у него внимание. Отсчёт разрешён спекой §4 явно
  (тренировка, а не проверка предсказания) — в отличие от экспозиции, где он запрещён.
- **Посекундного отсчёта внутри фазы (4-3-2-1) нет**: движение круга уже кодирует прогресс,
  вторая бегущая цифра — то, за чем начинают следить вместо дыхания.
- Шапки, `×` и подсказок нет. Выход называет факт — «Закончить».

**Состояния:**

- Идёт — круг дышит, хаптика по §5.
- Пауза — круг замирает в текущем масштабе, подпись «Пауза», хаптика стоит, отсчёт стоит,
  правая кнопка становится «продолжить». **`wasPaused = true`**.
- Конец — `HapticFeedback.success()`, переход на «После».
- Досрочно — «Закончить» пишет фактическую длительность и **ведёт на «После»**, а не
  закрывает флоу (1.5): вопрос «успел расслабиться» осмыслен и для короткой сессии.
- Приложение убито — запись уже в сторе, `actualDurationSeconds` пуст. Правдивое
  «начата и не завершена».
- Уход в фон — паузу не ставит (решение 11).
- Своего состояния ошибки нет: сохранение произошло на «Начать».

**Плохой день:** 0 тапов, экран доехал сам и до конца ничего не требует.

## 8. Экран «После»

**Работа:** записать один факт — успел расслабиться или нет. Главное действие — «Готово».

```
                                              (×)      закрыть; сохраняет частично
        Квадратное · 10 мин                            тихая мета, факт сессии

        Успел расслабиться?                            .h2, единственный вопрос
        ┌──────────┐  ┌──────────┐
        │    Да    │  │   Нет    │                     один тап
        └──────────┘  └──────────┘
        + Добавить заметку                             строка, НЕ раскрытое поле

   ┌───────────────────────────────────┐
   │             Готово                │                ← ЕДИНСТВЕННЫЙ акцент
   └───────────────────────────────────┘
```

**Фон:** `AppColors.sessionSurface` — остаётся тёмным. Вспышка светло-серого сразу
после практики попадает в максимально расширенный зрачок; возврат к светлому
происходит при закрытии флоу, на естественной границе.

**Компоненты:** да/нет — `SegmentedChoice(style: .segments)` (документированный кейс
«да/нет»), в тёмном варианте; заметка — паттерн `noteSection` из
`SituationalExposureFormView` (грабли по скроллу вычесаны там в комментариях) +
`AppTextEditor` в тёмном варианте; «Готово» — `PrimaryButton`.

**Чего здесь нет:** похвалы за завершённую сессию, итоговой статистики, **строки
правила лестницы** (решение 13).

**Состояния:**

- Дошла до конца — мета «Квадратное · 10 мин».
- Закончил раньше — мета называет факт без упрёка: «Квадратное · 4 из 10 мин».
  Не «Прервано», не «Не завершено».
- Не отвечено — «Готово» неактивно; да/нет — единственные данные экрана.
- Заметка — по умолчанию строка, раскрывается по тапу (спека §4 явно: «НЕ раскрытое поле»).
- Выход через `×` — `savePartial`, `didRelax` остаётся пустым. Данные, а не отмена (1.5).
- Ошибка — `.alert` «Не удалось сохранить. Попробуй ещё раз.»

**Плохой день:** 2 тапа здесь, **4 тапа на весь флоу**: плитка → Начать → Да → Готово.
Клавиатура только если сам раскрыл заметку.

| `ScrollingTickRuler` | `TickTrack.swift` | Маркер неподвижен по центру, лента едет под ним, риски равной высоты (`majorEvery: nil`). **Снап строго по ступеням** — промежуточных значений нет, иначе ломается правило лестницы. `HapticFeedback.selection()` на каждый стоп |
| `BreathingTypeCards` | `BreathingComponents.swift` | Ряд из трёх компактных карточек (иконка + название), выбранная залита `accent.opacity(Opacity.mediumBackground)`. Описание не внутри карточки — оно одно, для выбранного типа, в верхнем блоке |
| Тёмные варианты | `Components.swift` | `SegmentedChoice(.segments)`, `AppTextEditor`, `SessionFlowBottomPill` (описан как «чёрная капсула» — на тёмном исчезнет). Невыбранное — светлая заливка малой непрозрачности вместо `cardBackground`; выбранное — сплошной `accent`, как было |
| Обновить `USAGE.MD` | — | Грамматика заливок для **тёмной поверхности** — иначе каждый следующий тёмный экран изобретёт её заново. Плюс новые компоненты и снятие TODO про 4-ю фазу |

### Reduce Motion — разделяется чисто

- **Масштаб по фазам остаётся всегда.** Это инструмент упражнения, а не переход;
  остановить его = удалить инструмент.
- **Морфинг и дрейф выключаются.** Они амбиентные — ровно то, ради чего настройка есть.

Так исключения из кодекса §6 не требуется вовсе.

### Что проверяется только на устройстве

Размытие + непрерывная перерисовка **до 15 минут подряд** — реальная цена по батарее и
нагреву, `.blur` в SwiftUI дорогой. Закладываем `.drawingGroup()` и низкую частоту
морфинга. Если греется: первым падает дрейф, вторым радиус размытия, **масштаб фаз не
трогаем никогда**.

## 13. Definition of done

- От плитки «Дыхание» до дышащего круга **один тап**, если ничего не менять.
- Круг дышит по фазам, замирает на задержках, форма плавает; масштаб фаз читается
  однозначно (морфинг его не съедает).
- Хаптика по грамматике §5, **проверена на устройстве** — не в симуляторе.
- Экран не гаснет во время сессии; после выхода авто-блокировка восстановлена.
- Пауза работает и пишется в запись; обратный отсчёт идёт вниз.
- Ответ «успел/не успел» одним тапом, заметка спрятана за строкой.
- Выход в любой момент — включая убийство приложения — оставляет правдивую запись,
  а не отменяет её.
- Правило лестницы посчитано, покрыто тестами и показывается строкой на «До».
- Сессия пишется в SwiftData **на реальном устройстве** — собирает настоящие данные.

## 14. Что вышло не по плану при реализации

| Пункт плана | Как сделано | Почему |
|---|---|---|
| Тёмные варианты `SegmentedChoice`, `AppTextEditor`, `SessionFlowBottomPill` | **Не понадобились** для первых двух | Экран форсит `.environment(\.colorScheme, .dark)` — токены сами разрешаются в тёмные значения. Параметризовать пришлось только `SessionFlowBottomPill`: его чёрная капсула на тёмном исчезает |
| `.drawingGroup()` на кляксе | **Не поставлен** | Ставить вслепую нечего: `.blur` и так уходит в offscreen, а `drawingGroup` на меняющемся каждый кадр контенте может стоить дороже. Решается замером на устройстве |
| Отдельный движок фаз | Живёт **на самом `BreathingPattern`** | Фазы — свойство паттерна, отдельный тип был бы пустой обёрткой |
| Отсчёт конца сессии через `onChange` | Опрос в `.task` (250 мс) | Пауза двигает конец: таймер, заведённый на старте, пришлось бы сносить и пересоздавать на каждой паузе |
| Строки `Box breathing`, `Finish` | Перезаписаны переводы из 1.x | Лежали как «Квадратное дыхание» и «Завершить»; в новом флоу — «Квадратное» (в карточку не влезает) и «Закончить» (называет факт) |
| Раскладка «До»: описание сверху, контролы под ним | **Контролы прижаты вниз**, описание центрируется в остатке | Линейка и карточки типа — то, до чего дотягиваются большим пальцем; наверху им делать нечего |
| Раскладка «После»: мета и вопрос слева сверху | **Мета — в тайтл, вопрос — по центру обеих осей, ответы вниз** | Экран задаёт одну вещь; раскладка должна говорить это раньше текста |
| Иконка 4-6 | `rectangle` вместо стрелок | Тройка стала одной семьёй: квадрат (равные фазы) → прямоугольник (выдох длиннее) → круг (цикл без углов) |
| Круг: только масштаб | + размытый край, mesh-переливы, амплитуда 0.42→1.0 | Запрос автора после первого прогона; чёткая граница читалась как кнопка |
| «Done» на тёмном | Зелёный, а не инвертированный белый | Белая плашка — самое громкое, что можно положить на эту поверхность, и прилетает она сразу после практики. Выбранный сегмент Yes/No при этом **приглушённо**-зелёный, чтобы громкий акцент остался один |
